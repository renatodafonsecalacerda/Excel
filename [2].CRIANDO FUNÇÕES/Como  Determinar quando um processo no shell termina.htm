    Como: Determinar quando um processo no shell termina

    Office 2013
    Este tópico ainda não foi avaliado como - Avalie este tópico

    Última alteração: quarta-feira, 29 de junho de 2011

    É aplicável ao: Access 2013 | Office 2013

    Quando você executa a função Shell em um procedimento do VBA (Microsoft Visual Basic for Applications), ele inicia um programa executável de maneira assíncrona e retorna o controle ao procedimento. Esse programa em shell continua a ser executado independentemente do procedimento até ser fechado.

    Se o procedimento precisa aguardar o término do processo em shell, você pode usar a API do Windows para consultar o estado do aplicativo, mas isso não é muito eficiente.Este tópico explica um método mais eficiente.

    A API do Windows integrou funcionalidade que permite ao aplicativo aguardar a conclusão de um processo em shell. Para usar essas funções, você precisa ter uma identificador do processo em shell. Para isso, use a função CreateProcess no lugar da função Shell para iniciar seu programa em shell.

    Criando o processo em shell

    Para criar um processo endereçável, use a função CreateProcess para iniciar o aplicativo em shell. A função CreateProcess fornece ao programa o identificador do processo em shell por meio de um de seus parâmetros transmitidos.

    Aguardando a conclusão do processo em shell

    Depois de usar a função CreateProcess para obter um identificador do processo, você pode passar esse identificador para a função WaitForSingleObject. Isso fará com que o procedimento do VBA suspenda a execução até a conclusão do processo em shell.

    As etapas a seguir são necessárias para criar um procedimento do VBA que use a função CreateProcess para executar o aplicativo Bloco de Notas do Windows. Este código mostra como usar as funções CreateProcess e WaitForSingleObject da API do Windows para aguardar a conclusão de um processo em shell antes de continuar a execução.

    A sintaxe da função CreateProcess é complexa; portanto, no código de exemplo ela está encapsulada em uma função chamada ExecCmd. ExecCmd utiliza um parâmetro, a linha de comando do aplicativo a ser executado.

    Crie um módulo padrão e cole as seguintes linhas na seção Declarações:

    VBA
    Copiar 
    Option Explicit 
 
Private Type STARTUPINFO 
 cb As Long 
 lpReserved As String 
 lpDesktop As String 
 lpTitle As String 
 dwX As Long 
 dwY As Long 
 dwXSize As Long 
 dwYSize As Long 
 dwXCountChars As Long 
 dwYCountChars As Long 
 dwFillAttribute As Long 
 dwFlags As Long 
 wShowWindow As Integer 
 cbReserved2 As Integer 
 lpReserved2 As Long 
 hStdInput As Long 
 hStdOutput As Long 
 hStdError As Long 
End Type 
 
Private Type PROCESS_INFORMATION 
 hProcess As Long 
 hThread As Long 
 dwProcessID As Long 
 dwThreadID As Long 
End Type 
 
Private Declare Function WaitForSingleObject Lib "kernel32" (ByVal _ 
 hHandle As Long, ByVal dwMilliseconds As Long) As Long 
 
Private Declare Function CreateProcessA Lib "kernel32" (ByVal _ 
 lpApplicationName As Long, ByVal lpCommandLine As String, ByVal _ 
 lpProcessAttributes As Long, ByVal lpThreadAttributes As Long, _ 
 ByVal bInheritHandles As Long, ByVal dwCreationFlags As Long, _ 
 ByVal lpEnvironment As Long, ByVal lpCurrentDirectory As Long, _ 
 lpStartupInfo As STARTUPINFO, lpProcessInformation As _ 
 PROCESS_INFORMATION) As Long 
 
Private Declare Function CloseHandle Lib "kernel32" (ByVal _ 
 hObject As Long) As Long 
 
Private Const NORMAL_PRIORITY_CLASS = &H20& 
Private Const INFINITE = -1&

    Cole o seguinte código no módulo:

    VBA
    Copiar 
    Public Sub ExecCmd(cmdline As String) 
 Dim proc As PROCESS_INFORMATION 
 Dim start As STARTUPINFO 
 Dim ReturnValue As Integer 
 
 ' Initialize the STARTUPINFO structure: 
 start.cb = Len(start) 
 
 ' Start the shelled application: 
 ReturnValue = CreateProcessA(0&, cmdline$, 0&, 0&, 1&, _ 
 NORMAL_PRIORITY_CLASS, 0&, 0&, start, proc) 
 
 ' Wait for the shelled application to finish: 
 Do 
 ReturnValue = WaitForSingleObject(proc.hProcess, 0) 
 DoEvents 
 Loop Until ReturnValue <> 258 
 
 ReturnValue = CloseHandle(proc.hProcess) 
End Sub 
 
Sub Testing() 
 ExecCmd "NOTEPAD.EXE" 
 MsgBox "Process Finished" 
End Sub

    Digite Testing na janela Imediata. O Bloco de Notas será iniciado. Quando você fechá-lo, uma caixa de mensagem o notificará de que o processo foi concluído.
